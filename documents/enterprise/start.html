Начало работы с сервером

Подключитесь к серверу в режиме "Панели управления" (см. раздел "Подключение сервера") и изучите ее состав.

Панель управления сервером

Панель управления имеет модульную структуру. Каждый модуль управляет сервером определенного типа. Например существуют модули для управления гипервизором, почтовым сервером, контроллером домена, файловым сервером. Также, один модуль может управлять несколькими серверами, в случае если нужно создать комплексное интегрированное решение, в котором сервера должны взаимодействовать друг с другом и изменения, внесенные в параметры одного сервера должны распространяться на другие. Модуль не видит разницы, где находится сервер, которым он управляет. Сервер, которым управляет модуль Панели управления  может находиться на одном компьютере вместе с Панелью управления или на отдельном компьютере. Модуль панели управления воспринимает виртуальные сервера так же, как сервера, размещенные на отдельных компьютерах и одинаково управляет ими. Единственное, что нужно знать модулю Панели управления, это IP-адрес сервера или серверов, которыми он управляет.




На этой схеме изображен LVA Enterprise Server с панелью управления, включающей три модуля. Модуль "Управление гипервизором" предоставляет доступ к средствам управления самого LVA Enterprise Server, то есть управляет локальным сервером, на котором сам находится. Модуль "Управление контроллером" управляет виртуальным сервером с IP-адресом 192.168.0.5. Модуль "Управление почтовым сервером" управляет отдельным физическим сервером, который находится по адресу 192.168.0.6.

Каждый модуль представлен в виде отдельной закладки Панели управления:



При щелчке левой кнопкой мыши по закладке, открывается соответствующий модуль. Функции модуля представлены в виде дерева. Далее, перемещаясь по дереву модуля, вы даете ему различные команды, в ответ на которые модуль связывается со своим сервером по IP-адресу и выполняет их. Поэтому очень важно, чтобы модуль "знал" IP-адрес своего сервера и если IP-адрес сервера каким-то образом изменится, нужно дать об этом знать Панели управления, иначе она потеряет связь с сервером и команды перестанут выполняться. Чтобы изменить IP-адрес, используя который модуль Панели управления связывается с сервером, нужно:

1. Нажать правой кнопкой мыши по закладке этого модуля

2. Выполнить команду "Изменить IP-адрес" из контекстного меню



3. В появившемся окне изменить IP-адрес модуля



4. Нажать кнопку ОК

Если вы изменили IP-адрес сервера с помощью средств самого модуля управления им, не нужно выполнять описанных выше действий. Модуль их выполнит самостоятельно.

Стандартные модули

В своей базовой конфигурации, LVA Enterprise Server поставляется с одним модулем, который называется "Гипервизор". С его помощью вы можете создавать виртуальные сервера, а также менять некоторые базовые настройки самой системы (например, пароль пользователя root, настройки сети и др.). Однако на практике в большинстве случаев LVA Enterprise Server поставляется с уже предустановленными решениями, выполненными в виде виртуальных серверов и в Панели управления присутствуют закладки для управления ими.

В данном руководстве пользователя мы рассматриваем работу только со средствами модуля "Гипервизор". Работа с остальными модулями рассматривается в документации по серверам, которыми эти модули управляют. Документация всегда идет в комплекте с серверами.

Модуль "Гипервизор"

Этот модуль является по большей части группирующим интерфейсом к стандартным средствам управления, установленным в LVA Enterprise Server



  • Сервер - управляет локальной операционной системой,  установленной в LVA Enterprise Server. Позволяет, например, изменить пароль пользователя root, IP-адрес сервера, посмотреть состояние жестких дисков или  среднюю загрузку системы. Для целей такого управления используется стандартный интерфейс управления Webmin, который открывается в новом окне (или вкладке браузера) при нажатии левой кнопкой мыши по значку "Сервер"
  • Виртуальные машины - запускает подсистему управления виртуальными серверами, которые размещены на сервере. В качестве среды виртуализации LVA Enterprise Server использует VirtualBox. В качестве интерфейса управления виртуальными машинами используется интерфейс phpVirtualBox, который открывается в новом окне (или вкладке браузера) при нажатии левой кнопкой мыши по значку "Виртуальные машины". С помощью этого модуля можно создавать новые виртуальные машины, устанавливать операционные системы в них, включать, выключать или перезагружать их, смотреть что изображено на их экранах. Можно изменять параметры каждой виртуальной машины: выделять для них оперативную память, процессоры, создавать для них виртуальные жесткие диски, подключать созданные виртуальные жесткие диски, а также подключать к машинами образы CD и DVD-дисков в формате ISO, которые будут восприниматься операционными системами, установленными в виртуальных машинах как реальные накопители соответствующих типов.
  • Панель управления - содержит средства управления самой Панелью управления. В этом разделе можно изменить пароль администратора панели управления (admin), добавить других администраторов, если нужно, а также обновлять панель управления с сайта ЛВА.
  • Документация - содержит электронный вариант руководства пользователя, которое вы сейчас читаете.
Работа с подсистемами "Сервер" и "Виртуальные машины" рассматривается в следующих двух разделах этого руководства. Раздел "Панель управления" описан ниже.

Настройка панели управления

Раздел "Панель управления" включает подраздел "Администраторы", который позволяет изменять существующие учетные записи администраторов панели управления сервером, а также создавать новые. Изначально заведена одна учетная запись администратора: admin с паролем 111111, от имени которой происходит подключение к панели управления в первый раз. Рекомендуется как можно быстрее изменить пароль администратора.

Также с помощью этого раздела выполняется обновление системы через Интернет.

Создание новой учетной записи администратора

1. Разверните раздел "Панель управления", появится группа "Администраторы"



2. Нажмите правой кнопкой мыши на группе "Администраторы" и выполните команду "Новый пользователь" из контекстного меню



3. Появится окно для ввода данных учетной записи нового администратора



4. Требуется ввести "Имя" администратора, его "Пароль" дважды и "Конфигурационный файл".
  • Имя должно содержать только английские символы
  • Пароль может содержать любые символы.
  • И имя и пароль чувствительны к регистру, например, "admin" и "Admin" считаются разными пользователями
  • Конфигурационный файл используется системой для создания профиля пользователя для Панели управления. Нужно ввести путь к нему относительно корневого каталога панели управления. Вводится в следующем формате: /etc/WAB2/config/<имя>.conf.
Также можно выбрать из списка тему оформления и язык интерфейса панели управления. Тема оформления определяет цветовую гамму и другие элементы дизайна панели управления.

Увидеть новую тему можно будет при следующем входе в панель управления от имени этого пользователя. Также, если изменить "язык интерфейса", то он изменится при следующем входе в панель управления или при перезагрузке страницы в браузере.

Изменение параметров учетной записи администратора

Можно изменить имя и пароль любого, уже заведенного администратора. Существует два способа:

1. Разверните группу "Администраторы" и щелкните левой кнопкой мыши по нужной учетной записи. Появится окно с данными учетной записи, такое же, как показано на прошлом изображении. Отредактируйте данные и нажмите "OK".

2. Разверните группу "Администраторы", щелкните правой кнопкой мыши по нужной учетной записи и выполните команду "Изменить" из контекстного меню:



Появится окно с данными учетной записи, такое же, как показано на прошлом изображении. Отредактируйте данные и нажмите "OK".

Удаление учетной записи администратора

1. Разверните группу "Администраторы"
2. Нажмите правой кнопкой мыши на учетной записи администратора, которую хотите удалить
3. Выполните команду "Удалить" из контекстного меню


4. Система спросит: "Вы действительно хотите удалить пользователя ?". Если не передумали, ответьте "Да".

Обновление системы

Панель управления сервером это конфигурация, созданная компанией ЛВА с помощью платформы "ЛВА Конструктор Web-приложений 2". На момент приобретения, на сервере установлена самая новая исполняющая среда платформы "ЛВА Конструктор Web-приложений 2" и самая новая конфигурация панели управления сервера LVA Enterprise Server и всех модулей, которые требуются для управления виртуальными серверами.

Однако специалисты компании постоянно совершенствуют и саму платформу и конфигурации, созданные на ее основе и выпускают пакеты обновлений. При выходе очередного обновления, компания ЛВА делает почтовую рассылку всем пользователям решений на базе платформы "ЛВА Конструктор Web-приложений 2". В рассылке приведен список того, что обновилось в системе. В этом списке может быть описание новых функций, появившихся в системе, либо описание исправленных ошибок.

Чтобы обновить свою панель управления до самой новой версии нужно чтобы сервер был подключен к Интернет. Чтобы выполнить обновление нужно:

1. Войти в Панель управления
2. Нажать правой кнопкой мыши на разделе "Панель управления" и выполнить команду "Проверить обновления" из контекстного меню.



3. Система покажет окно "Будет выполнена проверка наличия обновлений ..."
4. После нажатия на кнопку "OK" система проверит, есть ли на сервере ЛВА обновления и если есть, предложит установить их, отобразив диалоговое окно "Появилась новая версия системы. Обновить сейчас ?". Если же новых версий не обнаружено, то появится окно "Система находится в актуальном состоянии".
5. Если обновления появились и система получила утвердительный ответ на вопрос "Обновить сейчас ?", то будет выполнено сравнение версии панели управления и платформы, установленных на сервере с версиями, доступными для обновления. После этого система загрузит и установит все необходимые обновления в правильной последовательности.
6. После того как обновление выполнится, система перезагрузит панель управления в браузере. Номер версии в заголовке в окне браузера должен измениться.
При загрузке обновления из Интернет могут возникнуть различные проблемы, о чем система обязательно сообщит. При возникновении проблем нужно проверить соединение сервера с Интернет и повторить операцию еще раз.

Компания ЛВА призывает всех пользователей решений на базе "ЛВА Конструктор Web-приложений 2" писать по адресу info@lvacompany.ru о всех недочетах и ошибках, обнаруженных в процессе работы с Панелью управления, а также любые пожелания, связанные с панелью управления (какие еще возможности хотелось бы иметь и как хотелось бы улучшить существующие возможности).
