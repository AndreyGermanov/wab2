06.04.2013

  • Создан реестр кодов классов сущностей для подсистемы передачи штрихкодов. У каждого класса есть свой код. Если он зашифрован в двумерном штрихкоде,  то он передается в качетсве параметра object_class в событии SCAN_CODE и может использоваться перехватчиками событий.
  • Добавлено событие SHOW_WINDOW, на которое реагирует окнонный менеджер. Он отображает окно с объектом, идентификатор которого передан в параметре object_id.
  • Реализован метод getBarCode, который вызывается файлом bc.php при получении штрихкода для класса объекта с номером, переданным в параметре "o". Класс объекта определяется по реестру кодов классов сущностей для подсистемы передачи штрихкодов. Этот метод по умолчанию генерирует событие SCAN_CODE, в которое передаются все параметры этого метода. Потомки могут переопределять метод getBarCode и выполнять другие действия при получении штрихода, например, заимодействовать со смартфоном, который передает штрихкод, задавать передающему вопросы.
  • Реализована реакция на событие SCAN_CODE на уровне менеджера окон. Менеджер окон при получении события проверяет, "слушает" ли это событие какой-либо другой объект данного класса и если не слушает, то сам реагирует на это событие, открывая окно данного объекта. Таким образом в системе появляется возможность поиска и отображения любого объекта по его двумерному штрихкоду.
  • Добавлена библиотека для генерации двумерных штрихкодов

  • В таблицу сущностей добавлена кнопка "Показать штрихкод". Если выделить сущность и нажать на эту кнопку, откроется окно с двумерным штрихкодом этой сущности.

  • Добавлен новый тип Web-страниц: форма регистрации на сайте. Он унаследован от типа "Форма обратной связи" и представляет из себя форму регистрации пользователя на сайте. Кроме обычной функциональности формы обратной связи, эта форма включает обязательные поля "Логин", "Пароль", "Пароль еще раз", "E-mail", "E-mail еще раз". При сохранении проверяется корректность заполнения этих полей, а также, зарегистрирован ли уже пользователь с таким логином. После проверки, форма регистрации работает так же как и обычная форма обратной связи. Она доставляется по почте администратору и добавляется в панель управления. Администратор на ее основании может зарегистрировать пользователя и отправить ему оповещение об этом.
  • Добавлена авторизация на Web-сайте. Для этого добавлен новый тип раздела "Форма авторизации", которая работает по принципу формы обратной связи с двумя полями "Логин" и "Пароль" и кнопкой "ОК". При вводе логина и пароля, система проверяет его среди системных пользователей и если совпадение найдено, устанавливает переменную сеанса "user" в имя пользователя, указанного в форме. В случае если пользователь авторизовался, форма меняет свой вид и показывает приветствие для этого пользователя и кнопку "Выход", при нажатии на которую переменная "user" обнуляется и форма снова отображает поля "Логин" и "Пароль".

  • В форму элемента справочника "Пользователи" добавлен флажок "Пользователь панели управления", который по умолчанию включен. Следовательно, в случае если указан логин, пароль и включен этот флажок, то пользователь также добавляется в список системных пользователей Web-сервера и может использоваться для авторизации в системе управление.
