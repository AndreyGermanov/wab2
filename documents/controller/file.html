Введение

LVA Business Server это файловый сервер с возможностью управления доступом к данным, которые на нем размещены. Сервер может поставляться в различных модификациях и в зависимости от модификации он также может выполнять дополнительные функции, такие как управление сетью или резервное копирование данных.

Основные функции выполняемые сервером:
  • Создание общих папок для хранения данных и управление ими
  • Создание учетных записей пользователей и управление ими
  • Создание групп пользователей и управление ими
  • Разграничение доступа пользователей и групп к общим папкам
  • Создание на базе сервера контроллера домена
Дополнительные функции, выполняемые сервером:
  • Выполнение роли сервера сетевой информации (DHCP)
    • Базовый режим - выдача всем компьютерам динамических адресов из предопределенного диапазона и выдача других настроек, одинаковых для всех компьютеров
    • Расширенный режим - регистрация каждого компьютера на сервере. Это позволяет выдавать каждому компьютеру свои собственные настройки
    • Комбинированный режим - позволяет выдавать часть настроек из базового режима по умолчанию для всех компьютеров (например адреса DNS-серверов), а часть настроек персонифицированно для каждого компьютера (например IP-адрес)
  • Разграничение доступа компьютеров, которые заведены для DHCP-сервера к общим папкам и к серверу в целом на уровне межсетевого экрана
  • Ведение журнала событий, в котором фиксируются все операции, которые каждый компьютер выполняет с данными, хранящимися в общих папок.
  • Резервное копирование и восстановление данных, хранящихся на сервере
  • Возможность настроить "Сетевую корзину". Сетевая корзина это папка на сервере, в которую попадают все файлы, удаляемые из общих папок сервера и хранятся в ней определенное количество времени.
  • Антивирусная защита данных, хранящихся на сервере
  • Создание почтовых ящиков для пользователей (в режиме интеграции с сервером Mystix Collector MX)
  • Управление доступом компьютеров к Интернет (в режиме интеграции с сервером Mystix Bastion ACS)
Большинство из этих функций работают в базовом режиме сразу после установки сервера. Для получения дополнительных возможностей требуется настройка. Настройка может осуществляться с удаленной рабочей станции администратора в одном из трех режимов подключения. Основной режим это подключение по зашифрованному по протоколу SSL каналу, с помощью Web-интерфейса, доступ к которому можно получить с помощью web-браузера Mozilla Firefox.

Также, в отдельных конфигурациях может быть организован доступ в режиме удаленного рабочего стола. Для этой цели с помощью системы NX Server реализуется защищенный по протоколу SSH канал. Для доступа используется клиент FreeNX, который находится на сервере в общей папке utils. Этот клиент должен быть установлен на рабочей станции администратора.

Также, по тому же протоколу SSH можно удаленно подключиться к командной строке сервера.

В этом руководстве пользователя будет показано как запустить сервер для работы в базовой конфигурации и как более тонко настраивать каждую функцию сервера.
